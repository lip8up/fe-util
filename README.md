# fe-util

前端工具库

## 原则

- 不要将与业务逻辑有关的代码放在本库
- 若 lodash 等优秀开源库中已有相应函数，直接使用之，不要重复造轮子
- 所有工具方法，都要提供单元测试代码

## 库构建、发布

每次发布之前，打开 `package.json`，修改 `version` 字段，然后执行：

```bash
yarn pub
```

## 单元测试

单元测试会在提交时自动执行，若不通过，则阻止提交，若要手动执行，运行：

```bash
yarn test
```

## 文档构建

文档会自动构建，若要手动构建，请执行：

```bash
yarn doc
```

文档生成在项目根目录下的 `doc`，用浏览器打开 `doc/index.html` 即可
